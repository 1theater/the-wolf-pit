# Волчья яма

Камерная пьеса-триллер в трёх действиях (рабочее название) и статический сайт-читалка для неё.

Внедрённый агент заперт в одной комнате с бандой после провалившегося налёта. Банда ищет наводчика — но единственный несомненный предатель в комнате тот, кто, спасаясь, назначает виновным другого. Тема — вина выжившего.

**Читать онлайн:** https://1theater.github.io/the-wolf-pit/

## Что в репозитории

### Тексты (контент)
| Файл | Что это |
|---|---|
| `the-wolf-pit.md` | Полный текст пьесы: разогрев, пролог, три действия, финал, эпилог. Открывается по умолчанию при заходе на сайт. (Третье действие пока заглушка — `...`.) |
| `role-book.md` | Актёрский разбор всех шести персонажей: сверхзадача, биография из намёков, отношения, тело/голос, ключевые сцены, ловушки роли. Для подготовки к репетициям. |
| `staging.md` | Постановочная часть: сценография, реквизит-лист, континуити крови и ран, световая и звуковая партитуры. Для постановщика и цехов. |
| `images/poster.png` | Постер пьесы (в начале текста пьесы). Постеры глав — `warmup.png`, `prologue.png`, `act-one.png`, `act-two.png`, `finale.png`, `epilogue.png`. Для третьего действия постер `act-three.png` ещё не создан — когда появится, добавить `![](images/act-three.png)` под заголовком ДЕЙСТВИЕ ТРЕТЬЕ. |

### Сайт-читалка (веб-версия)
Статический сайт на GitHub Pages, который рендерит markdown-файлы в тёмную адаптивную читалку с меню. Читается с десктопа и телефона.

| Файл | Назначение |
|---|---|
| `index.html` | Оболочка страницы: шапка с меню и контейнер контента. |
| `assets/app.js` | Логика: читает `pages.json`, по адресу (`#play`, `#roles`) загружает нужный `.md` и рендерит его. |
| `assets/style.css` | Тёмная тема, типографика для чтения, адаптив. |
| `assets/markdown-it.min.js` | Библиотека рендера markdown, вшита в репозиторий (без внешних CDN). |
| `pages.json` | Список страниц сайта (порядок, заголовки, файлы). Первая страница открывается по умолчанию (сейчас — пьеса); поле `hideInNav: true` прячет страницу из меню. |
| `.nojekyll` | Отключает Jekyll, чтобы GitHub Pages отдавал файлы как есть (важно для чтения «сырых» `.md`). |

### Проектные документы
`docs/superpowers/` — внутренние документы (для читателя/зрителя не предназначены):
- `specs/` — дизайн-замыслы: концепция пьесы и дизайн сайта.
- `plans/` — планы написания пьесы и реализации сайта.

## Как устроен сайт

Рендер — **клиентский, без сборки**. GitHub Pages просто отдаёт статические файлы; `app.js` в браузере читает «живой» `.md` прямо из репозитория и превращает его в HTML. Поэтому **правишь `.md`, пушишь — сайт обновляется** (запросы идут с анти-кеш меткой, изменения видны сразу, а не через кеш CDN).

- **Хостинг:** GitHub Pages, публикация из ветки `main`, папка `/` (корень).
- **Адрес:** `https://1theater.github.io/the-wolf-pit/` (формат сайта проекта: `<owner>.github.io/<repo>`).
- Все пути в коде относительные — сайт живёт в подпапке `/the-wolf-pit/`.

## Как добавить новую главу (сцены, свет и т.п.)

1. Положить новый `.md`-файл в корень репозитория.
2. Дописать одну строку в `pages.json`:
   ```json
   { "slug": "scenes", "title": "Сцены", "file": "scenes.md" }
   ```
   (Картинки внутри страницы добавляются прямо в `.md` через `![](images/файл.png)`.)
3. Закоммитить и запушить — пункт появится в меню сайта.

## Запуск локально

Открывать через локальный сервер, **не** через `file://` (браузер блокирует `fetch` локальных файлов):

```bash
python -m http.server 8000
# затем открыть http://localhost:8000/
```

## Хронометраж (черновая оценка)

Технические прикидки длительности по тексту пьесы. Модель: речь — 150 слов/мин, проигрывание действия из ремарок — 110 слов/мин, паузы — короткая 3 c, длинная/«Молчание»/«Тишина» 6 c. Это нижняя инженерная оценка; точное время даёт только репетиция — на сцене обычно дольше.

| Сцена | ≈ время |
|---|---|
| РАЗОГРЕВ | ≈ 1 мин |
| ПРОЛОГ | ≈ 1 мин |
| ДЕЙСТВИЕ ПЕРВОЕ · «Возвращение» | ≈ 7 мин |
| ДЕЙСТВИЕ ВТОРОЕ · «Наводчик» | ≈ 12 мин |
| ДЕЙСТВИЕ ТРЕТЬЕ · «…» | — (в работе) |
| ФИНАЛ · «Волчья яма» | ≈ 3 мин |
| ЭПИЛОГ | ≈ 1 мин |
| **Итого, без антракта** | **≈ 24 мин** |

> Цифры посчитаны вручную по тексту; при правках пьесы их нужно пересчитывать. Целевая длительность — 60 мин, то есть материал нужно нарастить примерно вдвое.

## Лицензия и использование

Тексты пьесы и книги ролей — авторские материалы постановки. Уточняйте условия использования у владельца репозитория.
